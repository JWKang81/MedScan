為了讓這套系統具備實用性並保護使用者隱私，除了核心的影像辨識外，建議規劃以下功能模組：

核心功能（第一階段建議實作）

使用者帳號管理：註冊、登入與登出功能。個人醫療資料具備高度隱私，必須透過帳號隔離資料（建議後續可使用 JWT 進行 API 驗證）。

影像上傳與處理：接收前端傳送的藥袋影像，進行格式驗證與儲存（存入伺服器或雲端空間），並將檔案路徑傳遞給辨識模組。

辨識結果確認與修改：影像辨識與 OCR（光學字元辨識）難免有誤差，系統必須將初步辨識結果回傳給前端，允許使用者在正式存檔前，手動校正藥品名稱、用法、劑量與天數。

個人用藥紀錄（歷史藥單）：將確認無誤的藥單資料與對應的藥品明細寫入 MySQL，提供使用者查詢過往的看診與用藥清單。

進階功能（後續擴充方向）

個人化服藥提醒：系統根據辨識出的「用藥頻率」（如：一天三次、飯後服用），轉換成時間排程，提醒使用者吃藥。

過敏與禁忌症警示：使用者可在個人設定中註記過敏藥物，當新上傳的藥袋中含有該成分時，系統主動跳出警告。

藥品衛教資訊庫：串接公開的藥品資料庫（如健保署資料），讓使用者點擊藥名就能看到外觀、副作用與適應症。

Flask + MySQL 後端基礎建置
在使用 Python-Flask 擴充時，建議一開始就將目錄結構切分清楚，這對後續整合第二部分的「影像辨識模組」會很有幫助。

1. 建議安裝的核心套件

Flask：輕量級後端框架。

Flask-SQLAlchemy：強大的 ORM 工具，讓你用 Python 物件導向的方式操作 MySQL，不用手寫長串 SQL 語法。

PyMySQL 或 mysqlclient：作為 Python 與 MySQL 溝通的底層驅動。
